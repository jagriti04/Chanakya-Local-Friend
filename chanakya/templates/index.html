Sandbox Guide

Available

  • Run Python and shell commands inside an isolated container.
  • Read host project files from read-only mounts at /host/repo and /host/chanakya_data.
  • Write persistent files only in the shared workspace for the current work id.
  • Retry after permission errors by copying files into /workspace first.
Writable path: /workspace -> chanakya_data/shared_workspace/<work_id> or temp

Blocked

  • Writing to host-mounted files or directories outside /workspace.
  • Direct host command execution, privilege escalation, or path traversal escapes.
  • Network-dependent downloads or external API calls from sandboxed code.

Common Permission Fix

  • If you see Permission denied or Read-only file system, the agent should copy the target file into /workspace and modify the copy there.
